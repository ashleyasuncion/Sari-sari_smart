PROJECT OVERVIEW
--------------------------------------------------------------------------------
Sari-Sari Smart is a mobile-first, simplified sales, inventory, and debt-tracking
application designed for traditional sari-sari store owners. The prototype is built
as a multi-page HTML application with shared CSS and JavaScript.

================================================================================
ARCHITECTURE
================================================================================

Multi-Page Structure:
  - 10 HTML files, each with its own phone frame
  - Shared style.css for consistent theming
  - Shared app.js with page-aware initialization

Navigation:
  - Bottom nav uses <a> tags for native page navigation
  - 5 main nav pages (sales, inventory, debts, reports, help) have bottom nav with bottom-nav id
  - 3 sub-pages (new_sale, add_product, new_debt) have NO bottom nav — only header with back + gear
  - Sub-pages highlight parent nav tab via pageToNav mapping
  - Settings gear passes ?from=PAGENAME for smart back-button destination
  - Sub-pages mapped through pageToNav (new_sale→sales, add_product→inventory, new_debt→debts)

State Management:
  - In-memory state object (state) with full application data
  - localStorage persistence across all pages (same origin)
  - Products, sales, customers, debts preserved between page loads

Localization:
  - Two languages: English (default) and Filipino
  - data-i18n attributes on all user-facing text
  - applyLanguage() function updates all text dynamically
  - Translation dictionary with 100+ keys per language

Data Flow:
  1. User completes sale on new_sale.html → sale saved to state + localStorage → redirect to sales.html
  2. Add/edit product on add_product.html → product saved → redirect to inventory.html
  3. Record debt on new_debt.html → debt saved → redirect to debts.html
  4. Record payment on debts.html overlay → payment saved → refresh debts list
  5. Settings changes on setting.html → saved to state → applied immediately

DESIGN
================================================================================

Visual Direction:
  - Primary: Green (#16a34a)
  - Background: Off-white (#f8faf9)
  - Surfaces: White with subtle borders
  - Accent: Warm yellow/amber (#f59e0b)
  - Danger: Muted red (#dc2626)
  - Typography: Inter font family
  - Rounded corners, soft cards, large touch targets
  - Clean, calm, uncluttered interface

Typography:
  - 3 text sizes (Standard 15px base, Large 17px, Extra Large 19px)
  - Sans-serif (Inter), good readability for older users

Phone Frame:
  - 393×852px (iPhone-like dimensions)
  - Realistic notch and home bar
  - Dark border with shadow for desktop display
  - Responsive scaling for smaller viewports

Tutorial:
  - 8-step guided tour across all 5 main pages (sales, inventory, debts, reports, help)
  - Smart box positioning: auto-places tutorial box at top or bottom based on highlight location
  - Cross-page navigation with localStorage state persistence
  - Replayable from Help page

Layout:
  - .app-content has 16px horizontal padding for consistent edge spacing
  - H2 page titles removed (header already shows page name via JS)
  - Product cards use vertical layout (info on top, actions below) to handle variable-length text
  - .btn-card has white-space: nowrap to prevent text wrapping in buttons
